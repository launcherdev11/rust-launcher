# Установка и подключение плагинов

## Требования

- 16Launcher **2.1.0** или новее
- Плагин с `"api_version": 1` в `plugin.json`
- Права на запись в папку данных лаунчера

---

## Способ 1: через интерфейс (рекомендуется)

1. Запустите 16Launcher.
2. Перейдите в **Настройки** (иконка шестерёнки).
3. Откройте вкладку **«Лаунчер»**.
4. Прокрутите до секции **«Плагины»**.
5. Нажмите **«Открыть папку»** — откроется каталог `plugins`.
6. Скопируйте папку вашего плагина в этот каталог.
7. Нажмите **«Перезагрузить»** в секции плагинов.

Плагин появится в списке. Если манифест корректен — кнопка **«Включить»** станет активной.

---

## Способ 2: вручную

### Windows

```powershell
# Пример: копирование example-плагина
$plugins = "$env:APPDATA\16Launcher\plugins"
New-Item -ItemType Directory -Force -Path $plugins
Copy-Item -Recurse "docs\plugins\example-plugin" "$plugins\example-hello"
```

### Linux / macOS

```bash
PLUGINS_DIR="${XDG_DATA_HOME:-$HOME/.local/share}/16Launcher/plugins"
mkdir -p "$PLUGINS_DIR"
cp -r docs/plugins/example-plugin "$PLUGINS_DIR/example-hello"
```

Перезапустите лаунчер или нажмите «Перезагрузить» в настройках.

## Структура папки плагина

```
my-plugin/
├── plugin.json          # ОБЯЗАТЕЛЬНО — манифест
├── main.js              # Опционально — точка входа JS
├── icon.png             # Опционально — иконка 256×256 PNG
├── hooks/
│   └── pre_launch.json  # Опционально — декларативный хук запуска
└── locales/             # Опционально — будущее расширение i18n
    ├── ru.json
    └── en.json
```

### Обязательные файлы

| Файл | Назначение |
|------|------------|
| `plugin.json` | Метаданные, разрешения, хуки, UI |

### Опциональные файлы

| Файл | Назначение |
|------|------------|
| `main.js` | JavaScript-логика (указывается в `entry`) |
| `icon.png` | Иконка в менеджере плагинов |
| `hooks/pre_launch.json` | Статические JVM/game аргументы при запуске |

## Включение и отключение

Состояние хранится в:

```
%DATA%/16Launcher/plugins-state.json
```

```json
{
  "format_version": 1,
  "plugins": {
    "my-plugin": {
      "enabled": true,
      "config": { "key": "value" }
    }
  }
}
```

- **Отключение** не удаляет файлы плагина — только прекращает выполнение хуков и скрывает UI.
- **Конфигурация** сохраняется при отключении.

---

## Обновление плагина

1. Закройте лаунчер (рекомендуется) или отключите плагин в настройках.
2. Замените файлы в папке плагина (кроме `plugins-state.json` — он общий).
3. Увеличьте `version` в `plugin.json`.
4. Нажмите «Перезагрузить».

Если вы изменили `id` — это **новый** плагин; старая запись в `plugins-state.json` не переносится автоматически.

---

## Удаление плагина

1. Отключите плагин в настройках.
2. Удалите папку плагина из `plugins/`.
3. (Опционально) удалите запись из `plugins-state.json`.

## Отладка

### Плагин не появляется в списке

- Проверьте, что `plugin.json` лежит **непосредственно** в подпапке `plugins/`, а не вложен глубже.
- Проверьте JSON на синтаксические ошибки (запятые, кавычки).
- Убедитесь, что `api_version` равен `1`.

### Плагин в списке с ошибкой

В карточке плагина отображается `load_error` — типичные причины:

| Ошибка | Решение |
|--------|---------|
| `plugin.json не найден` | Добавьте манифест в корень папки плагина |
| `Некорректный id` | Используйте только `a-z`, `0-9`, `-`, `_` |
| `api_version` не совпадает | Обновите лаунчер или манифест |

### JavaScript не выполняется

- Убедитесь, что `"entry": "main.js"` и файл существует.
- Откройте DevTools лаунчера (если доступно в dev-сборке) — логи с префиксом `[Plugin:id]`.
- Проверьте, что объявлена `function register(api)`.

### Хуки запуска не срабатывают

- Плагин должен быть **включён**.
- В `plugin.json` в массиве `hooks` должен быть `"pre_launch"` или `"post_launch"`.
- Для JVM-аргументов нужно разрешение `"modify_jvm_args"`.
- Опасные флаги (`-cp`, `-javaagent` и т.д.) **блокируются** фильтром лаунчера.

### Логи backend

При запуске из терминала (`npm run tauri dev`) смотрите вывод с префиксами `[Plugins]` и `[Launch]`.

## Публикация плагина

Рекомендуемый формат дистрибутива:

```
my-plugin-1.0.0.zip
└── my-plugin/
    ├── plugin.json
    ├── main.js
    └── ...
```

Пользователь распаковывает архив в `plugins/`.
